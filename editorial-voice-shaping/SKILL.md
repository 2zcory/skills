# Editorial Voice Shaping

This skill is now a compatibility layer.

Prefer `content-architecture` for new voice, messaging, rewrite, and public-copy shaping work. Use this skill only when the user explicitly wants a narrow rewrite-only pass or when an older workflow still points here.

## Default rule

- If the task touches route role, section logic, messaging hierarchy, voice rules, or content structure, use `content-architecture`.
- If the task needs both diagnosis and rewriting, use `content-architecture`.
- Stay here only when the user wants a narrow copy rewrite on an already-stable structure.

## Narrow workflow

When used, keep the pass compact:

1. State the route job.
2. Diagnose the main copy pathologies.
3. Rewrite with:
   - job first
   - clarity before cleverness
   - authored voice without self-performance
4. Name residual risks.

## Boundary

Do not expand this skill back into the primary content surface. That role now belongs to `content-architecture`.
