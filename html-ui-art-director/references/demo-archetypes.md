# Demo Archetypes

Use the bundled demo artifacts as quality references, not as templates to copy blindly.
Each demo should map to a decision branch, not become the default answer.

## `assets/demo-artifact/`

Use when the task needs:

- an AI workspace or product dashboard feel
- split hero plus live preview composition
- product-first trust building
- shape-led branding instead of image-heavy storytelling

This demo proves:

- strong first-view hierarchy
- a clear hero versus support split
- restrained use of radius and shadows
- product preview as the dominant trust asset

Primary layout branch:

- `split decision`
- secondary support from `snapshot plus flow`

Do not use this demo as a base when:

- persuasion and editorial pacing matter more than live preview
- dense operator controls should dominate

## `assets/demo-editorial-launch/`

Use when the task needs:

- an editorial or launch-page feel
- serif-led hierarchy
- asymmetry and publishing-style pacing
- low-icon, low-card composition

This demo proves:

- modern layout can work without dashboard tropes
- imagery can be abstract and atmospheric
- large type can carry the composition when the message matters more than product density

Primary layout branch:

- `editorial sequence`
- secondary support from `asymmetric hero`

Do not use this demo as a base when:

- the product needs an operational rail
- the user is configuring or comparing dense information

## `assets/demo-calm-operator/`

Use when the task needs:

- a calm enterprise or operator-facing tool
- stable context with low-drama controls
- dense but disciplined information
- minimal image reliance

This demo proves:

- the skill can build modern tooling surfaces without falling into harsh dashboard clutter
- hierarchy can come from spacing, grouping, and rails rather than decorative cards
- quiet enterprise design can still feel authored

Primary layout branch:

- `rail and canvas`
- secondary support from `control cluster`

Do not use this demo as a base when:

- brand storytelling is the main job
- one hero message should dominate the viewport

## Selection rule

Choose the demo whose composition logic is closest to the task:

- workspace
- launch page
- product marketing
- tool surface
- editorial narrative
- calm operator tool

Adapt the hierarchy, asset strategy, and copy structure. Do not reuse the same visual voice for unrelated products.
