# Layout Selection Rubric

Choose layout from the screen job, not from habit.

## Step 1: classify the screen job

Pick the closest primary job:

- persuade
- compare
- decide
- operate
- monitor
- create
- configure
- orient

Then pick the dominant user posture:

- scanning
- committing
- comparing
- editing
- following a guided path

If the screen tries to do more than two of these at once, reduce scope before choosing layout.

## Step 2: score the likely archetypes

Score each candidate from `1` to `5` on:

- fit for the primary screen job
- support for the next user decision
- hierarchy strength in the first viewport
- mobile survivability
- resistance to generic card stacking

Use the archetype with the strongest total and the clearest first-view story.

## Archetype guidance

## Asymmetric Hero

Use when:

- one message or one action should dominate
- the screen sells belief before detail

Avoid when:

- the user primarily needs to operate or compare dense information

## Split Decision

Use when:

- the user must understand two states or a before/after relationship
- preview matters almost as much as copy

Avoid when:

- one side has weak content and becomes filler

## Rail And Canvas

Use when:

- a stable context rail improves repeated work
- the main surface changes while controls or context remain visible

Avoid when:

- the rail becomes a parking lot for low-value controls

## Snapshot Plus Flow

Use when:

- users need one summary followed by one next action
- the product should feel calm and directed

Avoid when:

- the summary turns into a shallow mini-dashboard

## Editorial Sequence

Use when:

- storytelling, belief, or framing is the main job
- the page should feel authored and paced

Avoid when:

- operations or configuration dominate the task

## Control Cluster

Use when:

- the user needs grouped controls close to the work object
- tool behavior matters more than promotional framing

Avoid when:

- controls overwhelm the main result

## Step 3: challenge the choice

Before finalizing, answer:

- what would the default generic layout be here?
- why is the chosen layout better?
- what would fail on mobile if the wrong archetype were used?

If the answer is weak, the skill is probably following taste instead of logic.
