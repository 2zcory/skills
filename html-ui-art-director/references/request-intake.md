# Request Intake

Before choosing a direction, reduce the request into a few decision inputs.

## Minimum intake

Capture:

- product type
- screen type
- primary user
- next user decision
- trust level needed
- content density
- available assets

## Product type

Pick the closest:

- marketing site
- product workspace
- operator tool
- admin console
- consumer app
- portfolio or editorial surface

## Screen type

Pick the closest:

- landing
- dashboard home
- detail page
- editor
- onboarding
- settings
- pricing
- command or workflow surface

## Trust level

Choose one:

- high trust required
- moderate trust required
- expressive and exploratory

High-trust screens should avoid playful styling that weakens credibility.

## Content density

Choose one:

- low density
- medium density
- dense

Dense does not mean cluttered. It means structure matters more than decorative mood.

## Asset availability

Choose one:

- logo available
- product screenshots available
- illustration or photography available
- no real assets yet

If no real assets exist, the skill should choose placeholder treatment deliberately rather than pretending the product has finished brand material.

## Default assumptions

If the user gives little detail, default to:

- medium trust
- medium density
- no real assets yet
- one primary action

Then state those assumptions briefly instead of hiding them.
