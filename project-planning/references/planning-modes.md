# Planning Modes

Use this file when choosing how deep the planning pass should be.

## Create

Use `create` when:

- the project lacks a real planning baseline
- the user wants a complete plan to review
- current work is active but not well-sequenced

Output should prioritize:

- current reality
- sequencing
- risks
- deferrals
- next decision boundary

## Review

Use `review` when:

- a plan already exists
- the user wants faults, missing logic, or stronger structure surfaced

Output should prioritize:

- what the plan gets right
- biggest planning faults
- what to tighten, defer, or remove
- bottom line

## Replan

Use `replan` when:

- evidence changed
- scope changed
- a workstream matured faster or slower than expected
- the old sequence is no longer the best one

Output should prioritize:

- what changed
- what remains valid
- what sequence should change now
- what can still stay deferred
